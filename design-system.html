Sistema de diseño

Sistema de diseño
Tours y Traslados by Xcaret.

Este documento define los tokens, componentes y reglas que sostienen el sitio de Tours y Traslados by Xcaret. La arquitectura queda preparada para escalar a otros hoteles de Tours y Traslados by Xcaret en el futuro, sin tocar los componentes.

Visión general

El sistema separa cada decisión visual en dos capas:

  • Tokens del sistema — radios, espaciados, sombras, tamaños tipográficos. Iguales para todos los tenants.
  • Tokens por tenant — colores, fuentes, logo, hero. Distintos para cada hotel.

Los componentes (botones, inputs, tarjetas) solo consumen tokens — nunca colores literales. Eso hace que toda la marca se re-pinte cambiando un atributo en la raíz del documento.

Arquitectura multi-tenant

El tenant se resuelve por dominio. Una vez resuelto, se aplica como atributo en <html> y el navegador re-pinta toda la página leyendo los tokens correspondientes.

// 1. Middleware resuelve el tenant por dominio reservas.excursiontravel.com → tenant = "aloft" // 2. Next.js inyecta el atributo <html data-tenant="aloft"> // 3. CSS lee los tokens correspondientes [data-tenant="aloft"] { --brand-primary: #0e5c66; --font-display: 'Outfit'; ... }

La lógica de negocio (tarifas, reservas, pagos) está aislada por tenant_id en PostgreSQL con Row-Level Security. Un tenant nunca ve datos de otro.

Cómo agregar un nuevo tenant

Tres pasos. Cero código nuevo. Cero diseño nuevo.

// 1. Crear el registro del tenant en la base de datos POST /admin/tenants { "slug": "nuevo-hotel", "domain": "reservas.nuevohotel.com", "name": "Nuevo Hotel", "defaultPickup": "scplace-cun", "defaultDropoff": "scplace-nuevo", "currencies": ["mxn", "usd"], "languages": ["es", "en"], "commissionPct": 12 } // 2. Subir el logo POST /admin/tenants/nuevo-hotel/assets logo.svg, hero.jpg // 3. Definir el bloque de tokens en theme/nuevo-hotel.css [data-tenant="nuevo-hotel"] { --brand-primary: #c2185b; --brand-accent: #ffb300; --brand-hero: linear-gradient(...); --font-display: 'Manrope'; } // LISTO. Sitio disponible en reservas.nuevohotel.com

Si el admin de Excursion tiene UI para esto (que la tendrá), agregar un tenant es llenar un formulario de 60 segundos.

Reglas y do/don't

Sí

  • Usar siempre tokens (var(--brand-primary), var(--space-4)).
  • Diseñar mobile-first: el 70%+ del tráfico es celular.
  • Targets táctiles ≥ 44pt en mobile.
  • Estados claros: hover, focus, disabled, loading, error, vacío.
  • Verificar contraste WCAG AA en cada combinación de tenant.

No

  • Hardcodear colores literales (#0e5c66) en componentes.
  • Inventar tamaños de espaciado o tipografía fuera de la escala.
  • Mezclar márgenes en px con rem aleatoriamente.
  • Diseñar en desktop primero y "ver qué pasa" en mobile.
  • Usar imágenes raster cuando se puede SVG.
